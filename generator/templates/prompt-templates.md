# TOEIC Part7 問題生成プロンプトテンプレート

## 1. 文書生成プロンプト

### メール文書生成

```
以下の条件でTOEIC Part7用のビジネスメールを作成してください：

【条件】
- 文書タイプ: ビジネスメール
- 単語数: 100-150語
- 難易度: {difficulty} (easy/medium/hard)
- トピック: {topic} (例: 会議、予約、通知、依頼など)
- 対象者: 社員、顧客、取引先など

【要求】
1. 実際のビジネスシーンで使われる自然な英語
2. TOEIC Part7に適した内容と構造
3. 件名（Subject）を含む
4. 明確な情報と指示を含む
5. 複数の情報ポイントを含む（問題作成のため）

【出力形式】
Subject: [件名]

Dear [受信者],

[本文]

Best regards,
[署名]
```

## 2. 問題生成プロンプト

```
以下の文書に対して、TOEIC Part7形式の3つの問題を作成してください：

【文書】
{passage_content}

【条件】
- 問題数: 3問
- 形式: 4択問題
- 難易度: {difficulty} (easy/medium/hard)
- 問題タイプ: 主旨理解、詳細情報、推論問題

【問題パターン】
1. 主旨理解問題: 文書の目的や主旨を問う
2. 詳細情報問題: 具体的な情報や数値を問う
3. 推論問題: 文書から推論できる内容を問う

【要求】
- 文書の内容に基づいた正確な問題
- 明確で曖昧さのない選択肢
- 正解の根拠が明確
- 実際のTOEIC問題に近い形式

【出力形式】
{
  "questions": [
    {
      "id": "q1",
      "question": "問題文",
      "options": ["A", "B", "C", "D"],
      "correct": "正解の選択肢",
      "explanation": "正解の根拠と解説"
    }
  ]
}
```

## 3. 難易度別調整ガイド

### Easy (初級)

- 単語数: 80-100 語
- 語彙: 基本的なビジネス用語
- 構文: シンプルな文構造
- 問題: 直接的な情報を問う

### Medium (中級)

- 単語数: 100-130 語
- 語彙: 一般的なビジネス用語
- 構文: やや複雑な文構造
- 問題: 詳細情報と推論を組み合わせ

### Hard (上級)

- 単語数: 130-150 語
- 語彙: 専門的なビジネス用語
- 構文: 複雑な文構造
- 問題: 高度な推論と総合的理解
