# TOEIC Part7 問題生成ワークフロー

## 🚀 効率的な問題生成の手順

### Step 1: 文書生成

1. **プロンプトテンプレート選択**

   - メール、広告、記事から選択
   - 難易度とトピックを指定

2. **AI に文書生成を依頼**

   ```
   プロンプト例:
   "以下の条件でTOEIC Part7用のビジネスメールを作成してください：
   - 文書タイプ: ビジネスメール
   - 単語数: 100-150語
   - 難易度: medium
   - トピック: 会議の通知・変更"
   ```

3. **生成された文書の確認**
   - 長さの確認（100-150 語）
   - 自然な英語表現
   - 複数の情報ポイント

### Step 2: 問題生成

1. **生成された文書を AI に送信**

   ```
   プロンプト例:
   "以下の文書に対して、TOEIC Part7形式の3つの問題を作成してください：
   [生成された文書を貼り付け]"
   ```

2. **問題の品質確認**
   - 文書内容に基づいているか
   - 選択肢が明確か
   - 正解の根拠が明確か

### Step 3: データ形式変換

1. **JSON 形式に変換**

   ```json
   {
     "id": "passage4",
     "title": "",
     "type": "email",
     "content": "生成された文書内容",
     "questions": [
       {
         "id": "q1",
         "question": "問題文",
         "options": ["A", "B", "C", "D"],
         "correct": "正解",
         "explanation": "解説"
       }
     ],
     "metadata": {
       "difficulty": "medium",
       "estimatedTime": 300,
       "wordCount": 120,
       "questionCount": 3,
       "passageType": "email",
       "topic": "meeting_notification"
     }
   }
   ```

2. **passages.json に追加**
   - 既存の passages 配列に新しい問題を追加
   - ID の重複を避ける

### Step 4: 品質チェック

1. **文書チェック**

   - [ ] 適切な長さ
   - [ ] 自然な英語表現
   - [ ] 明確な情報構造
   - [ ] 複数の情報ポイント

2. **問題チェック**
   - [ ] 文書内容に基づいている
   - [ ] 選択肢が明確
   - [ ] 正解の根拠が明確
   - [ ] 難易度が適切

## 📊 量産のための効率化戦略

### バッチ生成

1. **複数の文書を同時生成**

   - 同じトピックで複数バリエーション
   - 異なる難易度で同じトピック

2. **テンプレート活用**
   - よく使われる文書パターンをテンプレート化
   - 変数部分のみ変更して生成

### 品質管理

1. **自動チェック**

   - 単語数チェック
   - 問題数の確認
   - 形式の統一

2. **人間による校正**
   - 自然さの確認
   - 難易度の調整
   - 文化的な適切性

## 🎯 推奨生成量

### 初期フェーズ（1-2 週間）

- **目標**: 20-30 問
- **内訳**:
  - Easy: 10 問
  - Medium: 10 問
  - Hard: 10 問

### 拡張フェーズ（1 ヶ月）

- **目標**: 50-100 問
- **内訳**:
  - メール: 30 問
  - 広告: 20 問
  - 記事: 20 問
  - その他: 30 問

### 完成フェーズ（2-3 ヶ月）

- **目標**: 200-300 問
- **多様性**:
  - 様々なビジネストピック
  - 異なる難易度レベル
  - 複数の文書タイプ
